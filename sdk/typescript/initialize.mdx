The initialization process is where you set the parameters for the SDK to work with.
This is where you set the network provider, alias, public address and your signer method.

You can simple call the `ClaveSDK` class with the parameters you want to use, or with empty parameters to be set in the future.

<CodeGroup>

```ts With valid parameters
import { ClaveSDK } from '@getclave/sdk';
import { Provider } from "zksync-web3"

const provider = new Provider("RPC_URL")

const claveSDK = new ClaveSDK({
    provider,
    alias: "USERNAME",
    publicAddress: "0x...000",
    messageSignerFunction: async (alias, message) => {
        // Sign the transaction with the alias and return the signature
        return "SIGNATURE"
    }
})
```

```ts With empty parameters
import { ClaveSDK } from '@getclave/sdk';
import { Provider } from "zksync-web3"

const claveSDK = new ClaveSDK()

// You can init when you have the parameters

const provider = new Provider("RPC_URL")

claveSDK.init({
    provider,
    alias: "USERNAME",
    publicAddress: "0x...000",
    messageSignerFunction: async (alias, message) => {
        // Sign the transaction with the alias and return the signature
        return "SIGNATURE"
    }
})
```

</CodeGroup>

## API

```ts
export type ClaveSDKInitParams = {
    /* Web3 Provider */
    provider: Provider;

    /* User's username */
    alias: string;

    /* User's 20 byte public address */
    publicAddress: string;

    /* Function that signs the messages */
    messageSignerFunction: (
        alias: string,
        transaction: string,
    ) => Promise<string>;

    logLevel?: ClaveSDKLogLevel;
};

```
